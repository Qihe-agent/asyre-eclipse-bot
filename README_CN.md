# Eclipse Bot

把你的 Discord 服务器变成一个有结构的 AI 工作台——上下文隔离、技能注入、人工控制生命周期。

---

## 为什么做这个

如果你用 AI Agent 做过任何正经事，你一定经历过退化问题。一开始，AI 很锐利，完美地执行指令，产出正是你想要的。然后你在同一个对话里又让它做另一件事。再一件事。到第 30 条消息，它开始混乱。到第 50 条，它开始产生幻觉——提到你从未说过的内容，把不同任务搅在一起，忘记你 20 条消息前设定的约束。

这不是 bug，这是大语言模型的工作原理。它们有一个上下文窗口——一段固定长度的文本是它们在任何时刻能"看到"的。你每发一条消息，就把更早的消息往后推。AI 并不真正"记住"任何东西——它每次都重新阅读可见的对话。当对话足够长，关键的指令就滑出了视野，AI 开始用听起来合理的虚构内容填补空白。

大多数团队的应对办法是手动开新对话。但这制造了另一个问题：上下文散落。你的设计讨论在一个对话里，跟进在另一个里，最终版本在第三个里，没人能找到任何东西。没有结构，没有生命周期，没有办法说"这个任务从这里开始，到这里结束"。

Eclipse Bot 就是为了同时解决这两个问题而存在的。

### 核心洞察：上下文隔离 + 技能注入

基本想法很简单但很强大：

**每个 AI 任务都有自己的隔离 thread，每个 thread 在创建时就自动注入了正确的技能上下文。**

当用户点击写作面板上的"开始"按钮时，Eclipse Bot 不是简单地创建一个空 thread。它创建 thread 后立即发送一条结构化的上下文消息，告诉 AI：

- 这是什么技能（例如"深度内容创作"）
- 用户选择了什么任务类型（例如"观点输出"）
- AI 应该做什么（完整的技能定义——流程、质量标准、输出格式）
- 有哪些约束条件

AI 带着清晰、全新的指令进入对话。它不需要猜自己应该做什么，不携带之前对话的包袱。它读取注入的上下文，等待用户输入，然后执行技能。

任务完成后，用户点击"归档"。AI 处理产出（总结、保存文件、发布——由技能定义决定），thread 关闭。干净的开始，干净的结束。

这不是理论上的改进。我们在 4 个 Discord 社群中运行这个系统，有 80+ 个活跃的技能面板。在有注入上下文的隔离 thread 里，AI 的产出质量与在无结构的聊天对话中相比，差距是巨大的。

### 为什么选 Discord？

合理的疑问。为什么不做一个自定义 Web 应用？为什么不用 Slack？

三个原因：

**1. 团队已经在 Discord 里了。** 对于社群、创意团队、小公司，特别是跟 AI Agent 协作的团队来说，Discord 已经是日常工作空间。建在 Discord 上意味着零迁移成本。用户不需要学新工具或在应用间切换。

**2. Thread 是完美的 AI 任务容器。** Discord thread 天然具备隔离性、可归档、可搜索，且嵌套在父频道下。它们支持富媒体（图片、文件、嵌入消息），有内置的成员管理。这恰恰是一个 AI 任务需要的容器——而 Discord 原生就提供了。

**3. Bot 生态已经成熟。** Discord.js 经过实战检验，API 文档完善，Bot 可以通过程序管理频道、权限、thread 和消息。Eclipse Bot 能用一条部署命令创建整个 AI 工作空间——频道、权限、面板，一步到位。

## 它到底怎么运作

### 技能面板系统

一个技能面板是发布在 Discord 频道中的一条常驻消息（嵌入消息 + 按钮）。每个频道专注于一种工作类型。面板提供四个按钮：

**开启** — 用户点击后（如果面板有类型定义）选择要启动的任务类型。然后 Eclipse Bot：
1. 创建一个带唯一编号的新 thread（如 `ASYR-042`）
2. 将 AI Bot 和相关团队成员加入 thread
3. 发送包含完整技能定义的上下文消息
4. 发送欢迎消息，告诉用户该做什么

AI Bot 会收到这样一条消息：

```
任务上下文：
- 技能：content-writing
- 类型：观点输出
- 说明：表达鲜明立场——说服+价值引导+颠覆

请读取技能文档，准备执行观点输出类型的工作。
等待用户发送主题。
```

这就是为什么 AI 不会产生幻觉或混乱。它在用户开口之前就确切知道自己要做什么。

**归档** — 任务完成后，用户回到主频道点击归档。Eclipse Bot 找到活跃的 thread，让用户选择要归档哪个。然后发送技能的归档指令给 AI，通常包括：
- 处理 thread 内容（总结、提取关键产出）
- 将文件保存到工作区
- 如果适用，发布结果
- 报告保存了什么、存在哪里

Thread 随后被标记为完成（名称前加上对勾），AI 不再在其中开展新工作。

**往期** — 发送查询给 AI，要求列出这个频道过去完成的任务。用于审计和查找之前的工作。

**清空** — 删除频道中所有非面板消息，保持工作区整洁。

### 为什么一个频道 = 一个技能

这是有意为之的架构决策，不是限制。

当你把多个技能放在同一个频道时，AI 看到的是不同主题的 thread 并列在一起。即使每个 thread 是隔离的，频道的视觉上下文也在混淆。用户搞不清该用哪个频道，新成员不知道该去哪里。

当每个频道只服务于一个技能时，整个频道就成了自文档化的。用户看到 `#deep-writing` 就知道这里干什么。他们看到 `ASYR-001 AI Agent 文章`、`ASYR-002 周刊` 这样的 thread——全部清晰地归属于写作。

这也意味着模板部署系统工作得很干净：每个面板 1:1 对应一个频道。部署一个模板，你就得到恰好数量的频道和恰好配置的面板。

### 生命周期模型

开启/归档循环是把这个系统和"直接用 Discord thread"区分开的关键：

```
[空闲] --> 用户点击开启 --> [活跃 Thread]
                                |
                         用户与 AI 协作
                         （消息、文件、图片）
                                |
                         用户点击归档 --> [处理并关闭]
                                              |
                                        AI 保存产出
                                        Thread 归档
                                              |
                                         [完成] --> 可搜索、可追溯
```

没有这个生命周期时：
- Thread 堆积，没有明确状态（这个做完了吗？还在进行中？被放弃了？）
- 旧 thread 中的 AI 上下文变得陈旧，但永远不会被清理
- 没有触发点让 AI 处理和保存产出
- 工作空间几周后就变得杂乱不堪

有了这个生命周期：
- 每个任务有明确的开始（开启）和结束（归档）
- 归档步骤强制 AI 产出一个最终交付物
- 关闭的 thread 被明确标记且可搜索
- 频道保持干净，为新工作腾出空间

这在团队协作中尤其重要。当多个人使用同一个频道时，生命周期让每个人都能清楚看到什么是活跃的，什么已经完成，什么需要关注。

## 模板系统

### 为什么需要模板

从零搭建一个 Discord AI 工作空间很繁琐。你需要：
1. 用正确的名称创建频道
2. 设置权限（私密频道需要特定的权限覆盖）
3. 设计面板嵌入消息（标题、描述、颜色、底部文字）
4. 定义技能类型（下拉菜单中出现哪些选项）
5. 编写归档指令（归档时 AI 做什么）
6. 编写往期查询指令（AI 如何查询过去的工作）
7. 配置按钮文字
8. 将面板发送到频道

一个面板需要 10-15 分钟。31 个面板的企业级部署需要好几个小时。

模板通过把所有这些决策打包到一个 JSON 文件来解决这个问题。一条命令读取模板，创建所有频道，设置所有权限，生成所有面板，并发布它们。一个需要几小时的 31 面板部署在几秒内就完成了。

### 6 个模板

这些不是凭空设计的。每个模板都是从真实生产部署中提取的，这些部署已经日常使用了数月。技能类型、归档指令和面板配置都经过了实际使用的打磨。

#### 1. 内容创作者（4 个面板）

适合写作者、KOL、内容团队，以及任何定期产出文字内容的人。

**深度写作** — 完整的内容创作流水线。用户从 7 种内容类型中选择（社会评论、深度科普、观点输出、教程干货、情感陪伴、揭秘爆料、自由主题），AI 接收完整的写作技能定义，包括意图分析、参数推导、质量评估标准和发布流程。这不是"帮我写篇文章"——而是一个结构化的 7 阶段写作流程，始终产出高质量内容。

**社媒排版** — 将 Markdown 文章转换为格式化的社交媒体卡片（1080x1440px PNG 分页）。支持文章排版、单页、图文并茂、长文预览和品牌模板。AI 处理图片生成、分页和视觉排版。

**AI 配图** — 使用类型×风格矩阵生成文章插图。用户选择信息图、场景图、流程图、对比图、框架图、时间线或智能混合。AI 分析文章内容并生成合适的插图。

**认知归档** — 记录对话、提炼洞察、构建知识库。四种类型：客户对话、内容探讨、认知升级和自由记录。归档流程包括原始保存、金句提炼、人物建档、评点，以及可选地从深度讨论生成长篇文章。

#### 2. 服务型公司（6 个面板）

适合设计公司、咨询公司、外包团队，以及任何管理客户工作的服务型企业。

**客户报价** — 基于行业模板生成专业报价。8 个行业预设（建筑、餐饮、制造、电商、教育、旅游、医疗、自定义），配有预配置的定价结构、范围定义和交付物列表。AI 了解行业背景，产出的报价看起来像是出自资深客户经理之手。

**客户跟进** — 使用诊断式销售（DBS）管理客户跟进流程。6 种跟进类型对应常见客户异议：价格焦虑、风险恐惧、价值模糊、决策拖延、需求漂移和常规回访。每种类型都有 AI 遵循的特定沟通策略。

**展示方案** — 生成落地页和展示材料。10 种类型，包括痛点调研、行业前瞻和 8 个行业特定模板。AI 产出完整的展示页面设计，包括文案、版面建议和视觉方向。

**设计工作台** — 通用设计工作台，支持排版、UI、品牌、海报、视频和其他设计工作。AI 协助设计方向、素材生成和迭代。

**公众号发布** — 管理微信公众号文章发布。两种模式：推送文章（排版、定时、发布）和账号管理。

**内容铸卡** — 将内容转换为视觉卡片格式。6 种类型：长图、信息图、多卡套装、视觉笔记、漫画和白板图。

#### 3. 创意工作室（7 个面板）

适合艺术团队、创意社群和专注于创意生产的兴趣群体。

**公告系统** — 结构化公告系统，4 种类型：重要决定、一般公告、纪念日和开放讨论。保持社群沟通有序。

**画院** — AI 驱动的视觉艺术创作。3 种类型：AI 生图、修图和风格转换。AI 处理提示词工程、迭代和产出精修。

**文渊阁** — 文学创作工作空间。3 种类型：诗词、散文和小说。AI 提供针对文学形式定制的写作辅助、编辑和评点。

**乐坊** — 音频制作工作空间。2 种类型：语音合成（TTS）和音频制作。AI 协助脚本准备、声音选择和音频后期。

**影像司** — 视频制作支持。2 种类型：剧本写作和后期剪辑指导。AI 帮助故事结构、镜头规划和剪辑笔记。

**戏台** — 表演和戏剧工作空间。3 种类型：编剧、导演和评论。为剧团和表演社群设计。

**经筵** — 教育内容创作。3 种类型：公开课、研讨会和读书会。AI 帮助结构化教育内容、创建演示材料和促进讨论。

#### 4. 全业务企业（31 个面板）

适合希望全面数字化运营的中小企业。这是最全面的模板，涵盖 6 个部门 31 个面板。

所有面板使用 **auto 模式** — 没有预定义类型。用户用自然语言描述需求，AI 从频道上下文推断适当的操作。这之所以可行，是因为每个频道的范围都很窄（比如"发票管理"——AI 知道这个频道里的任何请求都跟发票有关）。

**战略部**（3 个面板）：规划·品牌、决策·数据分析、知识·培训

**营销部**（6 个面板）：文案·创作、公众号·小红书、广告·投放、客户画像、营销·文档、设计·视觉

**销售部**（4 个面板）：客户·企业、渠道·销售、报价·合同、日程·会议

**运营部**（5 个面板）：采购·管理、生产·管理、打样·跟单、仓储·物流、审批·工作流

**财务部**（6 个面板）：财务·文档、发票·管理、税务·社保、记账·流水、成本·利润、凭证·密钥

**行政部**（7 个面板）：人事·档案、人员·KPI、行政·文档、注册·材料、合同·管理、公司·资产、法务·知产

这个模板为创始人或运营主管设计，让团队每个职能都获得 AI 辅助。各部门负责人获得与其工作相关的频道，AI 自动适应每个领域。

#### 5. 知识管理（5 个面板）

适合研究团队、培训机构和建设内部知识库的公司。

**知识入库** — 从 5 种来源导入知识：网页、YouTube 视频、Twitter thread、PDF 和手动输入。AI 提取、总结并结构化内容入库。

**知识锻造** — 将原始知识转化为结构化模块。5 种输入类型：文章、音频转录、视频转录、书籍和自由形式。产出是模块化、可复用的知识单元。

**思维工具** — 8 种认知框架的工具箱：圆桌讨论（多视角分析）、写作引擎、降秩引擎（简化）、概念解剖、投资分析、白话引擎、论文阅读和旅行研究。每种框架引导 AI 执行特定的思维方法论。

**工作区整理** — 文件系统维护。2 种模式：完整扫描（审计所有文件）和生命周期清理（归档旧文件、整理新文件）。

**管控台** — 面板管理界面。3 种类型：部署面板、重发面板和面板审查。这是管理面板系统本身的元面板。

#### 6. 生活社群（8 个面板）

适合兴趣社群、生活方式社群和社交空间。

**养生堂** — 健康养生指导。3 种类型：养生方案、食疗和功法。

**药房** — 传统医药工作空间。3 种类型：药方、药材辨识和问诊。

**情书阁** — 浪漫写作辅助。2 种类型：情书和表白。

**月光亭** — 反思空间。2 种类型：夜话和冥想引导。

**许愿池** — 社群许愿空间。3 种类型：许愿、祈福和感恩。

**音乐花园** — 音乐鉴赏。2 种类型：推荐和共赏。

**棋室** — 策略游戏。1 种类型：对弈。

**私密空间** — 个人日记。2 种类型：日记和私密条目。

### 混合和自定义模板

模板之间不互斥。你可以给同一个服务器部署多个模板：

```bash
# 先部署内容创作
node templates/deploy-template.cjs templates/content-creator.json \
  --guild 123 --bot 456 --brand "My Studio"

# 之后追加知识管理
node templates/deploy-template.cjs templates/knowledge-hub.json \
  --guild 123 --bot 456 --brand "My Studio"
```

也可以排除不需要的面板：

```bash
# 部署服务型公司，但不要公众号和内容卡片
node templates/deploy-template.cjs templates/service-agency.json \
  --guild 123 --bot 456 --brand "Agency" \
  --exclude wechat_pub,visual_card
```

你还可以通过复制和修改 JSON 文件来创建自己的模板。Schema 很直观——每个面板都是一个独立的配置块。

## Eclipse Bot 如何与你的 AI 连接

这是一个重要的架构要点：**Eclipse Bot 不包含 AI 逻辑。** 它纯粹是 UI 和工作流管理层。它负责：

- 创建和管理 Discord 频道和 thread
- 发布带交互按钮的面板嵌入消息
- 将用户交互路由到正确的处理器
- 向新 thread 注入技能上下文
- 管理开启/归档生命周期

实际的 AI 工作由另一个 bot 完成——你的 AI bot。当 Eclipse Bot 创建 thread 时，它 @提及你的 AI bot 并发送技能上下文。你的 AI bot 读取上下文并开始工作。

这种分离是有意为之的：

**1. AI 无关性。** 你的 AI bot 可以是任何东西——OpenAI assistant、Claude agent、用 LangChain 或任何其他框架构建的自定义 bot。Eclipse Bot 不关心 AI 背后的技术。它只需要一个能读取 Discord 消息并回复的东西。

**2. 独立扩展。** 你可以升级、重启或替换你的 AI bot，而不需要动 Eclipse Bot。面板系统无论 AI 发生什么都保持运行。

**3. 多 AI bot。** 你甚至可以为不同频道配不同的 AI bot。内容频道用专门的写作 AI，业务频道用数据分析 AI，艺术频道用创意 AI。

典型的部署架构：

```
Eclipse Bot（本项目）           你的 AI Bot（独立项目）
         |                              |
   管理面板                        读取技能上下文
   创建 thread                     执行任务
   处理按钮                        在 thread 中回复
   注入上下文                      处理归档
         |                              |
         +--------- Discord -----------+
```

## 安装指南

### 前置要求

- Node.js 18+ 和 npm
- 一个 Discord bot token（[在这里创建](https://discord.com/developers/applications)）
- 一台服务器（任何 Linux VPS，或者你的本地机器用于测试）

### 第 1 步：创建 Discord Bot

1. 前往 [Discord 开发者门户](https://discord.com/developers/applications)
2. 点击"New Application"，起个名字
3. 进入"Bot"标签页，点击"Add Bot"
4. 复制 bot token（稍后需要）
5. 开启这些特权网关意图（Privileged Gateway Intents）：
   - **Message Content Intent** — 读取消息必需
   - **Server Members Intent** — 欢迎消息和成员追踪必需
   - **Presence Intent** — 状态更新必需
6. 进入"OAuth2" -> "URL Generator"
   - 选择 scopes：`bot`、`applications.commands`
   - 选择权限：`Administrator`（或手动选择：管理频道、管理消息、发送消息、创建公开 Thread、在 Thread 中发消息、嵌入链接、附加文件、读取消息历史、添加反应、使用斜杠命令）
   - 复制生成的 URL，在浏览器中打开以邀请 bot 加入你的服务器

### 第 2 步：克隆和安装

```bash
git clone https://github.com/yzha0302/eclipse-bot.git
cd eclipse-bot
npm install
```

### 第 3 步：配置环境

```bash
cp .env.example .env
```

编辑 `.env`：
```env
CLIENT_TOKEN="粘贴你的-bot-token"
CLIENT_ID="粘贴你的-bot-client-id"
DATABASE_URL="file:./dev.db"
```

`CLIENT_ID` 和 token 不同。你可以在 Discord 应用的"General Information"页面找到它，也叫"Application ID"。

### 第 4 步：配置 Bot 设置

```bash
cp src/bot-config.example.json src/bot-config.json
```

编辑 `src/bot-config.json`：
```json
{
  "bot": {
    "id": "粘贴你的-bot-client-id",
    "ownerId": "你的-Discord-用户-ID",
    "developers": ["你的-Discord-用户-ID"]
  },
  "defaults": {
    "brand": "My Bot",
    "color": 15105570,
    "footer": "My Bot"
  },
  "guilds": {
    "粘贴你的-服务器-ID": {
      "brand": "我的社群",
      "footer": "我的社群",
      "welcome": {
        "enabled": true,
        "channelId": "欢迎频道ID",
        "title": "欢迎！",
        "description": "欢迎来到 **{guildName}**，{displayName}！你是第 #{memberCount} 位成员。"
      },
      "team": {
        "mention": ["你的用户ID"]
      }
    }
  }
}
```

如何找到你的 Discord ID：在 Discord 中开启开发者模式（设置 -> 高级 -> 开发者模式），然后右键点击任何用户、频道或服务器，选择"复制 ID"。

### 第 5 步：构建和运行

```bash
npm run setup    # 初始化 SQLite 数据库
npm run build    # 编译 TypeScript 为 JavaScript
npm start        # 启动 bot
```

你应该看到：`Logged in as: YourBotName` 和 `Serving 1 guild(s)`。

生产环境使用 PM2：
```bash
npm install -g pm2
pm2 start dist/index.js --name eclipse-bot
pm2 save
pm2 startup    # 服务器重启时自动启动
```

### 第 6 步：部署技能面板

现在 bot 在运行了，但还没有面板。部署一个模板：

```bash
  --guild 你的服务器ID \
  --bot 你的BOT_CLIENT_ID \
  --brand "我的工作室" \

# 确认无误后正式部署

  --guild 你的服务器ID \

  --brand "我的工作室"
```

检查你的 Discord 服务器——你应该能看到新的频道和其中发布的面板。

## 内置社群功能

Eclipse Bot 不仅是 AI 工作流管理器，还包含完整的社群工具包：

### 等级系统

基于 Mee6 的 XP 公式：`XP = 5x^2 + 50x + 100`，其中 x 是当前等级。用户每条消息获得 15-25 XP（有防刷屏保护）。功能包括：等级卡、角色奖励、排行榜、XP 倍率、免 XP 频道和角色。

### 经济系统

跨服务器的双层货币系统：
- **金币** — 高级货币（1:1 美元价值），由管理员管理
- **银币** — 通过聊天（每条消息 0.5）和每日签到获得，加上升级奖励

命令：`/wallet`、`/daily`、`/transfer`、`/transactions`

### 工单系统

多分类支持工单，配有子分类、私密 thread 和团队通知。

### 欢迎系统

完全按服务器可配置的欢迎消息，支持变量替换和自动角色分配。

## 来源

Eclipse Bot 不是周末项目或概念验证。它从一个生产系统中提取，该系统在 4 个 Discord 社群中日常运行，拥有 80+ 个活跃技能面板。

技能面板架构、模板配置和开启/归档生命周期经过了数千次真实 AI 任务执行的打磨。6 个模板包含实际的生产配置——类型、指令和工作流都是基于真实用户反馈和真实 AI 产出质量迭代而来的。

开源的决定来自于看到太多团队在 AI 上下文管理这个问题上挣扎。解决方案不是更强大的 AI——而是围绕人与 AI 交互建立更好的结构。Eclipse Bot 提供的就是这个结构。
